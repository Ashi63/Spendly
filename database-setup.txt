     Plan: Step 1 ‚ÄĒ Database Setup

     Context

     Spendly is a Flask + SQLite expense tracker built as a step-by-step
     tutorial. Step 1 establishes the data layer: implementing database/db.py
     with three functions (get_db, init_db, seed_db) and wiring them into app.py
      so the database is ready on startup. All future steps (auth, expenses,
     profile) depend on this foundation.

     ---
     Files to Modify

     - database/db.py ‚ÄĒ implement all three functions (currently just comments)
     - app.py ‚ÄĒ add imports and startup initialization call

     ---
     Implementation Plan

     1. database/db.py

     get_db()
     - Opens spendly.db in the project root using sqlite3.connect()
     - Sets connection.row_factory = sqlite3.Row for dict-like row access
     - Executes PRAGMA foreign_keys = ON
     - Returns the connection

     init_db()
     - Calls get_db() to get a connection
     - Executes CREATE TABLE IF NOT EXISTS for both tables:
       - users: id (PK autoincrement), name (TEXT NOT NULL), email (TEXT UNIQUE
     NOT NULL), password_hash (TEXT NOT NULL), created_at (TEXT DEFAULT
     datetime('now'))
       - expenses: id (PK autoincrement), user_id (INTEGER, FK ‚Üí users.id, NOT
     NULL), amount (REAL NOT NULL), category (TEXT NOT NULL), date (TEXT NOT
     NULL), description (TEXT), created_at (TEXT DEFAULT datetime('now'))
     - Commits and closes connection

     seed_db()
     - Calls get_db()
     - Checks SELECT COUNT(*) FROM users ‚ÄĒ if > 0, returns early (prevents
     duplicates)
     - Inserts demo user: name="Demo User", email="demo@spendly.com", password
     hashed via werkzeug.security.generate_password_hash("demo123")
     - Gets the inserted user's id
     - Inserts 8 sample expenses linked to the demo user, covering all 7
     categories (Food, Transport, Bills, Health, Entertainment, Shopping,
     Other), with dates in YYYY-MM-DD format spread across the current month
     - Commits and closes connection

     All SQL uses parameterized queries (no string formatting).

     2. app.py

     - Add import at top: from database.db import get_db, init_db, seed_db
     - Before if __name__ == "__main__":, add startup block using Flask app
     context:
     with app.app_context():
         init_db()
         seed_db()

     ---
     Categories (exact values)

     Food, Transport, Bills, Health, Entertainment, Shopping, Other

     ---

     1. Run python app.py ‚ÄĒ app should start without errors, spendly.db file
     should appear in the project root

     from database.db import get_db

     print(list(db.execute("SELECT * FROM users").fetchall()))
     print(list(db.execute("SELECT * FROM expenses").fetchall()))
     2. Should show 1 user and 8 expenses
     3. Run python app.py a second time ‚ÄĒ no duplicate rows should appear
     4. Run pytest ‚ÄĒ all existing tests should still pass
